Task description

You are given a list of data entries that represent entries and exits of groups of people into a building. An entry looks like this:

{"timestamp": 1526579928, count: 3, "type": "enter"}

This means 3 people entered the building. An exit looks like this: {"timestamp": 1526580382, count: 2, "type": "exit"}

This means that 2 people exited the building. timestamp is in Unix time.

Find the busiest period in the building, that is, the time with the most people in the building. Return it as a pair of (start, end) timestamps. You can assume the building always starts off and ends up empty, i.e. with 0 people inside.

Data for task
Timestamp Count of people Type
1526579928 3 enter
1526580382 2 exit
Solution

First at all I need to sort all events by time.

Secondly, I need to add a column with the number of people in the building for each ofthe timestamps.

To fill this column I need to add incoming ('enter') and subtracting outgoing ('exit') peolpe.

In order to avoid the situation when there is a negative number of visitors in the building, I set the initial number of visitors equal to the sum of those who left the building.

In order not to have to scan the array thrice, I need to create a variable that will store the maximum number of people in the building for the current (itterated at the moment) timestamp.

Task description

Write a function to flatten a nested dictionary. Namespace the keys with a period.

For example, given the following dictionary: { "key": 3, "foo": { "a": 5, "bar": { "baz": 8 } } }

it should become: { "key": 3, "foo.a": 5, "foo.bar.baz": 8 }

You can assume keys do not contain dots in them, i.e. no clobbering will occur.
